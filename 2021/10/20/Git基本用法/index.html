Git基本用法

尽量写得精简~
Git 是一个分布式版本控制系统

版本控制 主动提交 中央仓库 这三个要素，共同构成了版本控制系统的核心
开发团队中的每个人向中央仓库主动提交自己的改动和同步别人的改动，并在需要的时候查看和操作历史版本，这就是版本控制系统。

版本控制系统 有 中央式版本控制系统 和 分布式版本控制系统

为了精简，这篇博客只讲分布式版本控制系统，即 git 这种操作

分布式版本控制

具备 中央仓库 和 本地仓库

一个团队的每个成员都有一个本地仓库。这个仓库里有所有的版本历史。成员在自己的本地仓库进行代码提交，然后在适当的时机把本地仓库的代码记录上传到中央仓库进行整合
