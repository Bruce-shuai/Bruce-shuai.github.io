interface

interface 不是一种类型，应该被翻译成 接口，或者说使用上面介绍的类型，创建一个我们自己的类型：

啥时候需要声明类型

理论上来说在我们声明任何变量的时候都需要声明类型（包括普通变量、函数、组件、hook 等等），声明 函数、组件、hook 等需要声明参数 和 返回值的类型。

但是在很多情况下，TS 可以帮我们自动推断，我们就不用声明了，比如：

.d.ts

JS 文件 + .d.ts 文件 === ts 文件。
.d.ts 一般是用于第三方库 打补丁用的。有些库为了方便更多用户使用没有直接用 ts 版本。但如果某些用户需要 ts 版本的这个库，可以下载该库的 ts 声明文件(前提是这个文件是别人已经写好的才行，写该库声明文件的人可以是原本库的作者或其他人)即 xxxx.d.ts 文件

.d.ts 文件可以让 JS 文件继续维持自己 JS 文件的身份，而拥有 TS 的类型保护。

一般我们写业务代码不会用到，但是点击类型跳转一般会跳转到 .d.ts 文件。

类型别名 type

// 类型别名 –> 给自定义的类型取一个名字
type jackFavoriteNumber = string | number

在很多情况下，类型别名和 interface 是可以互换的。二者主要的两个区别如下：

  1. interface 无法实现联合类型或交叉类型,但是 type 可以实现
  1. interface 没法实现 Utility Type

Utility Type

Utility Type 是用 Type 来实现的

Utility Type作用: 操作已有类型，生成新类型
Utility Type用法：即利用泛型给 Utility 传入一个其他类型，然后 Utility type 对这个类型进行某种操作
