前端面试之道-浏览器渲染原理

浏览器有两个关键工作——首先，把网络资源“取”回来。在这个过程里，浏览器不生产任何额外内容，它只是 Web 世界里的“搬运工”；然后，把这坨资源消化掉，像翻译官一样，把底层密密麻麻的鬼画符转化为用户喜闻乐见的可交互页面。

浏览器零部件

用户界面、浏览器引擎都是“领导”一般的存在：它们不做特别具体的工作，简直就是命令的搬运工
渲染引擎最为核心(现在 浏览器内核 === 渲染引擎)
渲染引擎，是一个浏览器的灵魂

浏览器内核分类

  • Trident（IE）
  • Gecko（火狐）
  • Blink（Chrome、Opera）
  • Webkit（Safari)

Blink 也是基于 WebKit 衍生出来的一个分支。因此，WebKit 无疑仍然是当下浏览器世界真正的“大 Boss”

渲染引擎工作流解析

Tips: 这块知识非常容易出问答题/作为性能优化面试题的切入点，大家需要引起重视。

浏览器的渲染引擎承载着把静态资源转换为可视化界面的任务

渲染引擎处理过程：资源文件 –> HTML 解析 –> CSS 解析 –> 样式结构合并 –> 布局 –> 绘制

1. HTML 解析 –> 对应产物：DOM 树
在这一步浏览器对 HTML 文档进行解析，并在解析 HTML 的过程中发出了页面渲染所需的各种外部资源请求。

2. CSS 解析 –> 对应产物：CSSOM 树
在这一步浏览器对 HTML 文档进行解析，并在解析 HTML 的过程中发出了页面渲染所需的各种外部资源请求。

3. 样式与结构合并 –> 渲染树
将样式信息和文档结构合并，最终生成页面 render 树（:after :before 这样的伪元素会在这个环节被构建到 DOM 树中)

4. 布局阶段 –> 盒模型
页面中所有元素的相对位置信息，大小等信息均在这一步得到计算。

5. 页面绘制 –> 目标界面
在这一步中浏览器会根据我们前面处理出来的结果，把每一个页面图层转换为像素，并对所有的媒体文件进行解码
