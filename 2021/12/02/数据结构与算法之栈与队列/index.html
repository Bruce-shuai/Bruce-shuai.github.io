数据结构与算法之栈与队列

前言：很多时候相关题目中压根不会出现“栈”、“队列”这样的关键字，但只要你深入到真题里去、对栈和队列的应用场景建立起正确的感知，那么很多线索都会在分析的过程中被你轻松地挖掘出来。

栈

有效括号

题目(力扣 155)：给定一个只包括 ‘(‘，’)’，’{‘，’}’，’[‘，’]’ 的字符串，判断字符串是否有效

此题利用栈，来验证是否括号有对称性

记一个规律： 题目中若涉及括号问题，则很有可能和栈相关

注意：此题看似简单，但特别容易考虑不周，导致出错…

每日温度问题

(此题理解我还不是特别深刻)

题目(739):根据每日气温列表，请重新生成一个列表，对应位置的输出是需要再等待多久温度才会升高超过该日的天数。如果之后都不会升高，请在该位置用 0 来代替。

例子:给定一个列表 temperatures = [73, 74, 75, 71, 69, 72, 76, 73]，你的输出应该是 [1, 1, 4, 2, 1, 1, 0, 0]。

栈结构可以帮助我们避免多余事情

避免重复操作的秘诀就是及时地将不必要的数据出栈，避免它对我们后续的遍历产生干扰
有趣的理念：递减栈

此题核心点：最小栈里放置的不是温度，而是温度的下标(便于记录温度的时间点)

“最小栈”问题

题目(力扣 155):设计一个支持 push ，pop ，top 操作，并能在常数时间内检索到最小元素的栈

核心：配置一个辅助栈！

队列

关于队列，在算法面试中大家需要掌握以下重点：

  • 栈向队列的转化
  • 双端队列
  • 优先队列

如何用栈实现一个队列

题目(力扣 232): 使用栈实现队列的下列操作
push(x) – 将一个元素放入队列的尾部。
pop() – 从队列首部移除元素。
peek() – 返回队列首部的元素。
empty() – 返回队列是否为空

示例：
MyQueue queue = new MyQueue();
queue.push(1);
queue.push(2);
queue.peek(); // 返回 1
queue.pop(); // 返回 1
queue.empty(); // 返回 false

核心点：栈转队列 需要两个栈!
