TS 知识总结

TS 的优势

使用 JS 的话，大部分的错误都是在 runtime(运行时)的时候发现的
TS 能够在静态代码中，就能找到其中的一些错误 -> 强类型

尽量在 ts 中少用 any 类型

如果有 ts 的类型错误，但是现在又不想去修改 可以在出现问题的上一行使用： // @ts-ignore 来压制错误

typeof

(ts 中的 typeof 和 js 中的 typeof 二者是不一样的， js 的 typeof，是在 runtime 时运行的，ts 中的 typeof 是在静态环境中运行的)
