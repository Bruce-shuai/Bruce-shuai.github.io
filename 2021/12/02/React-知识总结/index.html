React 知识总结

public 文件夹

我常常会使用 npx create-react-app '文件名' 来创建一个 react 项目。 这个脚手架工具帮助我们生成的基本项目中 会有 public 文件夹。这个文件夹是什么有什么用？

事实上，public 文件夹是不参与打包的，它里面包含了真正的静态文件。 但是 react 项目的 src 文件夹里的所有文件最终会被打包工具例如：webpack 打包成一个 bundle.js

public文件夹里的图片、资源、字体等等内容 都是为 public 里面的index.html文件服务的
mainifest.json是用来配置 pwa 的
robots.txt 是想告诉搜索引擎如何来对待我们的项目(哪些内容可以访问，哪些内容不可以访问)

现在的 react 项目不再需要手动引入 React 了

现在不需要手动引入 React 了即：import React, {...} from 'react' –> import {...} from 'react' 不过前提是当前这个文件里没有用到 React 对象的一系列内容才行…

我们的 jsx/tsx 文件是用的 babel 插件：@babel/plugin-transform-react-jsx 来转化成 js 代码的
新版本的@babel/plugin-transform-react-jsx新增了一些操作，让我们不必手动引入 React 了
