项目一 知识点总结+反思

(面试法宝之一)

项目一使用的内容

为什么我两个项目都在用 react hooks 因为未来的 react 18 就是完全面向 react hooks 而来的。我所做的项目 以及了解到的资料知道，hooks 其实能够胜任大多数的 class 工作。目前我好想就只知道错误边界这里不得不用 class 其余都没用到过 class。 而且 hooks 是灵活的，非常契合函数式编程的思维…我也无聊的时候会弄了一些 自定义的 hooks。

该项目的特点：

  • 以 TS 作为类型约束
  • 代码全为 react hooks
  • 自定义钩子
  • 使用 react router v6
  • 状态管理( redux toolkit、react query)
  • 单元测试
  • 任意的拖拽功能
  • context/react toolkit 来实现客户端的全局管理
  • react query 来实现服务端的全局管理
  • 乐观更新 –>

后面要修改，我会把 emotion 变为 tailwindcss 然后在样式方面有进一步的提升

本项目的后端 借助的是 npm 包 npm-jira-tool 分布式后端服务 开发者工具用 MSW 以 Service Worker 为原理实现了”分布式后端”

  1. 请求被 Service Worker 代理
  2. 后端逻辑处理后，以 localStorage 为数据库进行增删改查操作

每个用户的浏览器上都安装了一个独立的后端服务和数据库，再也不受任何中心化服务的影响 点击’清空数据库’便可以重置后端服务

  1. HTTP 请求精准控制 开发者工具可以精准地控制 HTTP 请求的时间、失败概率、失败规则

感觉这个项目的很多东西都过度封装了，就难以理解… 比如这个 useAsync 就不好用
