React Hooks的思考与总结

代码地址

React 的中文含义即：“反应”或“创新”，它描述了 React 这样一个前端框架的核心原理：当数据发生变化时，UI 能够自动把变化反应出来

JSX 仅是语法糖，意味着。你不用 JSX 的写法，同样能够写出 React 来

JSX 不是一个新的概念，而只是原生 JS 的另一种写法，但是这种写法，将会大大降低你上手 React 的难度

在 React 组件中，任何一个 state 发生变化时，整个函数组件其实是被完全执行一遍的， 而且除了 state，多次的执行之间没有任何关系。所以在考虑这样一个场景的实现时，我们 的思考方式就是要首先考虑这个组件有哪些状态(state)，这些状态的变化是由什么触发 的，从而将整个功能串联起来。

Hooks 的使用规则：1. 只能在函数组件的顶级作用域中使用，2. 只能在函数组件或者其他 Hooks 中使用

useState: 让函数组件具备维持状态的能力

比较 Class 组件的 state 和 hooks 组件的 state

二者最大的区别就是， 类组件的 state 只能有一个，然后再通过不同的属性来表示不同的状态。 而 useState 则可以很容易地创建多个 state，所以它更加的语义化

useEffect: 执行副作用

副作用是指一段和当前执行结果无关的代码
useEffect 中代码的执行是不影响渲染出来的 UI 的。

其实不必刻意把 useEffect 对应到某几个生命周期方法中，虽然我以前也尝试这样做过，但是，有了这样的思维后，就有点应试教育的那种感觉，不能完全拥抱 hooks。我现在对于 useEffect 的理解是每次组件 render 完后判断依赖并执行就可以了

注意一个小问题，useEffect 的 callback 要避免直接的 async 函数，需要封装一下：

useEffect 关于依赖项的一些规则：

  • 没有依赖项，则每次 render 完后都会重新执行
  • 空数组作为依赖项，则只在首次执行 render 后触发
  • 普通依赖项,第一次以及依赖项发生变化后执行
  • useEffect 还允许你返回一个函数，用于在组件销毁的时候做一些清理的操作

在写 hooks 时候的注意点：

  1. 在 useEffect 的回调函数中使用的变量，都必须在依赖项中声明 (这里的知识点有点重要)
  2. Hooks 不能出现在条件语句或者循环中，也不能出现在 return 之后
  3. Hooks 只能在函数组件或者自定义 Hooks 中使用

(当然，React 官方给我们提供了一个 ESLint 的插件，专门用来检查 Hooks 是否正确被使用，它就是 eslint-plugin-react-hooks)

useCallback: 缓存回调函数

当没必要的变量(一些函数)更新后导致组件重新渲染，有了 useCallback 就能让没必要的变量不引起组件的渲染

useMemo: 缓存计算的结果

可以避免重复的计算，对于性能的提升会有很大的帮助

useRef: 在多次渲染之间共享数据

useContext: 定义全局状态

如何理解函数生命周期

忘掉 Class 组件的生命周期…

在函数组件中你要思考的方式永远是：当某个状态发送变化时，我要做什么，而不是像在 Class 组件的某个生命周期方法中我要做什么

在函数组件中，是没有生命周期机制的
